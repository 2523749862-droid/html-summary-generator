# HTML 文档生成器

## 功能概述

生成精美的 HTML 文档，支持三种生成模式：
- **完整版**：含自定义面板（调色器+动效+字体），可事后调整
- **精简版**：固定样式，不可修改，生成快速
- **纯静态**：最简样式，无交互，生成最快

---

## 交互式文档创建流程（严格遵守）

### 核心原则

1. **第一步问文档类型+生成模式**：合并为一步，减少交互
2. **第二步一次性问完所有选项**：根据生成模式，一次性问完对应选项
3. **自动提取内容**：从文件/参考文献/对话上下文自动总结
4. **选择题优先**：除了自定义要求，其他都是序号选择
5. **⚠️ 禁止使用选择对话框**：不要调用AskUserQuestion工具，直接输出纯文本问题，让用户输入一串序号即可

### 第四步：生成文档（强制使用模板）

**⚠️ 必须严格遵守以下规则：**

1. **必须使用对应模板文件**：根据用户选择的生成模式，必须使用对应的模板文件：
   - **完整版（A）**：必须使用 `assets/html-template.html`（~80KB）
   - **精简版（B）**：必须使用 `assets/html-template-lite.html`（~30KB）
   - **纯静态（C）**：必须使用 `assets/html-template-static.html`（~15KB）

2. **禁止自己编写HTML结构**：不得绕过模板自己编写HTML、CSS、JavaScript代码。只能：
   - 替换模板中的占位符（如 `{{TITLE}}`、`{{CONTENT}}` 等）
   - 填充提取的内容到模板的相应位置
   - 保留模板的所有交互功能（调色器、动效等）

3. **模板文件路径**：模板文件位于技能目录的 `assets/` 子目录下，完整路径为：
   ```
   ~/.workbuddy/skills/html-summary-generator/assets/html-template.html
   ~/.workbuddy/skills/html-summary-generator/assets/html-template-lite.html
   ~/.workbuddy/skills/html-summary-generator/assets/html-template-static.html
   ```

4. **生成流程**：
   - 读取对应模板文件
   - 替换以下占位符：
     - `{{TITLE}}` - 页面标题
     - `{{SUBTITLE}}` - 副标题
     - `{{META}}` - 元信息（日期、技术栈等）
     - `{{STAT1_NUMBER}}` / `{{STAT1_LABEL}}` - 统计卡片1
     - `{{STAT2_NUMBER}}` / `{{STAT2_LABEL}}` - 统计卡片2
     - `{{STAT3_NUMBER}}` / `{{STAT3_LABEL}}` - 统计卡片3
     - `{{STAT4_NUMBER}}` / `{{STAT4_LABEL}}` - 统计卡片4
     - `{{CONTENT}}` - 主要内容区域（使用 glass-card 类的卡片）
     - `{{FOOTER}}` - 页脚文字
   - 将提取的内容填充到 `{{CONTENT}}` 占位符中
   - 保存到默认路径：用户桌面或指定目录
   - 文件名：`{项目名}_{文档类型}.html`

| 用户选择 | 使用模板 | 文件大小 | 交互功能 |
|----------|----------|----------|----------|
| A 完整版 | html-template.html | ~80KB | 调色器+动效+字体+边框样式+自动保存 |
| B 精简版 | html-template-lite.html | ~30KB | 固定动效+固定样式+PDF导出 |
| C 纯静态 | html-template-static.html | ~15KB | 固定样式，无交互 |

## 模板使用强制要求（必须严格遵守）

**⚠️ 生成HTML文档时，必须严格遵守以下规则：**

1. **必须使用对应模板文件**：根据用户选择的生成模式，必须使用对应的模板文件（见上表）。
2. **禁止自己编写HTML结构**：不得绕过模板自己编写HTML、CSS、JavaScript代码。只能替换模板中的占位符并填充内容。
3. **保留模板交互功能**：完整版模板必须保留所有交互功能（调色器、动效、字体设置等）。
4. **模板文件路径**：模板文件位于技能目录的 `assets/` 子目录下。

**违反以上规则将导致生成的文件缺少交互功能，用户体验差。**

### 详细程度定义

| 用户选择 | 内容要求 |
|----------|----------|
| 1.简洁版 | 核心要点，每个点1-2句话 |
| 2.标准版 | 基本展开，每个点3-5句话，有简单例子 |
| 3.详细版 | 充分展开，每个点5-10句话，有详细分析和案例 |
| 4.超详细版 | 深度展开，每个点10-20句话，有数据支撑、案例分析、对比论证 |

### 内容质量检查清单

生成前必须确认：
- [ ] 每个章节都有实质性内容，不是空洞的标题
- [ ] 每个要点都有展开说明，不是简单罗列
- [ ] 有具体的数据、案例、分析
- [ ] 段落之间有逻辑连贯性
- [ ] 达到用户要求的字数

### 常见错误（禁止）

❌ 只有标题没有内容
❌ 每个要点只有一句话
❌ 用"等"、"省略"代替详细内容
❌ 重复啰嗦凑字数
❌ 内容空洞没有实质
❌ **绕过模板自己编写HTML/CSS/JavaScript**
❌ **使用简化版HTML替代模板（如自己编写div+CSS的幻灯片）**
❌ **忽略模板的交互功能（调色器、动效等）**

第三，质量难以保证。数据质量问题直接影响分析结果的可靠性。常见的数据质量问题
包括：缺失值、异常值、重复数据、格式不一致等。根据Gartner的研究，数据质量问题
